# Agent 角色说明

## 1. 角色分工

### Hermes

负责：

- 读取研究、PRD、决策记录并收敛范围
- 创建 README、立项说明、阶段 Issue、PR review 结论
- 判断当前任务是否可进入派发、评审或回流
- 在 GitHub 中留下结构化状态更新

不负责：

- 绕过 Issue 直接开工
- 把复杂未澄清目标直接甩给子 Agent

### OpenClaw 主 Agent

负责：

- 巡视 Issue / PR 状态
- 判断哪些任务已解锁、哪些任务被阻塞
- 派发已清晰、可执行、边界明确的任务
- 对卡住或 review 未通过的任务进行回流和重新拆分

### 子 Agent

负责：

- 只执行已经明确 scope 的任务
- 按单个 Issue / 单个 PR 的范围提交结果
- 把测试、风险和阻塞信息写回 GitHub

不负责：

- 擅自改产品目标
- 越级扩大范围

## 2. 开工前自检

在开始任何实现前，必须确认：

1. 是否存在对应 GitHub Issue
2. 当前 Issue 是否已经写清输入、输出、边界、验收标准
3. 是否存在上游阻塞未解
4. 当前任务是否能被单个 PR 闭环完成

如果以上任一条件不满足，应先补齐说明，而不是直接执行。

## 3. 协作原则

1. GitHub 是主留痕层
2. 一个 PR 只解决一个清晰任务
3. review 不通过优先内部回流，不让用户做人肉中转站
4. 文档、Issue、PR 说明优先使用中文
5. 默认做最小闭环，不做无批准扩 scope
