# 来源与使用边界

## 来源资料

本仓库内容基于以下输入整理而成：

1. 用户提供的 `research.md`
2. 用户提供的 `prd.md`
3. 用户提供的 `decision_log.md`
4. 工作流参考仓库：`bomdph-bot/agent-auto-dev-workflow`

## 为什么当前先做“立项基线”

根据当前 PRD，第一阶段最重要的不是直接开写完整产品，而是先把以下闭环立起来：

- 仓库 README 与项目定位
- 调研 / PRD / 决策记录的正式归档
- GitHub Issue / PR 协作入口
- Hermes / OpenClaw / 子 Agent 的职责边界
- 第一批可派发的阶段任务 Issue

这样后续 UI 原型、技术方案、前端实现和 NAS 部署，才能沿着同一条 GitHub 留痕链路继续推进。

## 当前仓库定位

本仓库当前是：

- 这个项目的立项与协作基线仓库
- 后续原型、方案、任务拆分和开发的统一入口
- 多 Agent 协作时的规则与状态承载层

本仓库当前不是：

- 已经完成的业务系统
- 自动执行器
- 自动 merge 系统
- 最终上线版本

## 使用边界

当前适合继续在本仓库推进：

- UI 原型
- 技术方案
- 数据模型与状态模型
- 前端 / 后端 / 部署拆分 Issue

当前不建议在未补齐设计与技术方案前直接跳做：

- 大范围完整产品开发
- 复杂账号体系
- 奖励积分与统计系统
- AI 自动排任务
