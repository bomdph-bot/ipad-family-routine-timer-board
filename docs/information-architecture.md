# 页面信息架构与状态图

> 关联 Issue：#3
> 目标：把孩子主界面、家长后台以及 3 种基础状态的展示内容整理成可执行的信息架构，为后续 UI 原型提供结构边界。
> 依据：`docs/prd.md`、`docs/decision-log.md`

---

## 1. 页面总览

| 页面 | 归属 | 主要职责 | 是否 MVP 必需 |
|------|------|----------|--------------|
| 孩子主面板 | 孩子端 | 展示时间条、任务格子、自由时间、当前下一步；支持任务完成操作 | ✅ |
| 家长后台 | 家长端 | 配置孩子档案、任务、时长、顺序、时间窗口；支持撤销纠偏 | ✅ |
| 孩子切换入口 | 孩子端 | 让孩子选择自己的身份（头像/昵称），进入对应面板 | ✅（顶部切换条） |
| 开始前待机状态 | 孩子端 | 显示"今晚计划将于 xx:xx 开始"，时间条为未消耗状态 | ✅ |
| 进行中状态 | 孩子端 | 实时倒计时、任务格子、自由时间发光、紧迫感反馈 | ✅ |
| 结束后未完成状态 | 孩子端 | 显示"已超出计划时间"、保留未完成任务；不直接消失或空白 | ✅ |

> 注：开始前 / 进行中 / 结束后是**孩子主面板的 3 种子状态**，不是独立页面。

## 2. 孩子端 — 页面结构与展示内容

### 2.1 孩子主面板（Child Main Panel）

**位置：** iPad 主屏幕，是孩子每次打开 app 看到的首个界面。

**顶部 — 身份切换条**

- 显示当前孩子昵称 + 颜色/头像
- 4 个孩子的切换入口（可左滑或点击展开）
- 切换后面板内容整体刷新为该孩子的任务视图
- 不显示其他孩子的任务

**中部 — 主时间条（核心视觉区）**

- 横轴代表当晚起止时间（如 5:30pm–9:00pm）
- 任务格子按顺序和时长占据对应位置（深色/实心块）
- 自由时间区域为未被任务占据的空白区间，以**发光效果**显示
- 随时间流逝，右侧剩余时间区域逐渐缩短
- 任务完成后，对应格子变淡/划线，时间条中该任务区域转化为发光自由时间

**任务格子区域（时间条下方）**

- 当前下一步任务：**高亮边框 + 显著标签"现在做这个"**，在其他任务之上
- 其他待完成任务：普通样式，顺序展示
- 已完成任务：划线 / 淡出样式，不占据倒计时关注区
- 未完成且时间紧迫：发热/挤压/强调效果（视觉紧迫感）

**关键文案提示**

- 进行中：在时间条上方显示"剩余 X 分钟"
- 开始前：时间条上方显示"今晚计划将于 5:30 开始"
- 结束后未完成：时间条上方显示"已超出计划时间"，任务区域保留未完任务

**底部 — 操作入口**

- 任务完成按钮：点击该任务即标完成（无二次确认，或可选轻量确认）
- 不展示复杂管理操作

---

### 2.2 状态一：开始前（Pre-Start）

**触发条件：** 当前时间还未到达当晚计划起始时间。

**界面表现：**

- 孩子主面板整体显示，但主倒计时**未进入消耗状态**
- 时间条完整展示今晚所有任务格子（可预先查看）
- 顶部文案：**"今晚计划将于 xx:xx 开始"**（温和提示，不施压）
- 无紧迫感反馈
- 可看到自由时间预计总量（预估发光区域）

**禁止出现：**
- ❌ 空白页面
- ❌ 报错"无数据"
- ❌ 开始真正的倒计时

---

### 2.3 状态二：进行中（In Progress）

**触发条件：** 当前时间在计划起始时间与结束时间之间。

**界面表现：**

- 主倒计时实时消耗，时间条从左到右推进
- 当前下一步任务**高亮**，其他任务次之
- 自由时间区域：已获得的可自由活动时间，以发光效果呈现
- 紧迫感触发：当 `剩余时间 < 未完成任务总时长`，未完成任务格子产生轻微挤压/发热/强调效果，风格仍为儿童友好
- 孩子可点击任务旁的完成入口标记完成

**禁止出现：**
- ❌ 隐藏当前下一步任务的高亮
- ❌ 自由时间用普通灰色块表示（必须发光）

---

### 2.4 状态三：结束后未完成（Ended — Incomplete）

**触发条件：** 当前时间已超过计划结束时间，且存在未完成任务。

**界面表现：**

- 时间条推进到末端，右侧区域显示"已超时"
- 顶部文案：**"已超出计划时间"**（直接说明，不回避）
- 未完成任务**保留显示**，允许孩子继续完成
- 不直接跳转到空白或完成页面（孩子不会感到"界面消失了"）
- 不展示新的倒计时（时间条停止）

**禁止出现：**
- ❌ 超时后直接消失或跳到成功页（除非全部完成，见下）

---

### 2.5 边界状态：全部完成（Ended — All Done）

**触发条件：** 计划结束时间前，所有任务已标记完成。

**界面表现：**

- 时间条完整且几乎全部转化为发光自由时间
- 顶部文案：**"今晚任务已完成，剩余时间自由安排"**
- 无紧迫感
- 可继续在面板上停留

---

### 2.6 边界状态：无任务（No Tasks Configured）

**触发条件：** 家长未配置当天任何任务。

**界面表现：**

- 显示"今晚暂无任务安排"
- 不显示空白报错或技术错误信息

---

## 3. 家长端 — 页面结构与展示内容

### 3.1 家长后台（Parent Admin Panel）

**入口：** 从孩子主面板可通过管理员入口（锁 / 设置图标）进入家长后台；或直接访问独立 URL。

**核心原则：** 只处理"配"和"改"，不让孩子主界面承载管理功能。

---

### 3.2 后台子页面列表

#### 3.2.1 孩子管理页（Children Management）

**职责：** 管理 4 个孩子的档案。

**字段：**
- 孩子昵称
- 颜色/头像标识
- 排序顺序

**操作：**
- 添加孩子（最多 4 个）
- 编辑昵称/颜色
- 删除孩子（谨慎操作，需二次确认）

---

#### 3.2.2 今晚计划页（Tonight's Plan）

**职责：** 设置当晚起止时间、任务列表及分配关系。

**字段：**
- 当日起始时间（默认 5:30pm，支持当天调整）
- 当日结束时间（默认 9:00pm，支持当天调整）
- 任务列表（见下方任务配置）

---

#### 3.2.3 任务配置页（Task Configuration）

**职责：** 创建和编辑任务，设置时长、顺序、所属。

**每个任务的字段：**
- 任务名称
- 预计时长（分钟）
- 顺序（1、2、3…）
- 任务性质：共用 / 指定孩子
- 指定孩子（当任务性质为个人时选择）

**操作：**
- 新增任务
- 编辑任务
- 删除任务
- 拖拽调整顺序

---

#### 3.2.4 进度监控页（Progress Monitor）

**职责：** 查看 4 个孩子各自的完成状态，支持撤销纠偏。

**展示内容：**
- 4 个孩子的任务完成情况（以卡片或列表形式）
- 每个孩子的已完成 / 未完成 / 已超时状态
- 最近一次完成操作记录（用于撤销）

**纠偏操作：**
- 撤销最近一次完成：恢复任务状态，时间条回退
- 不提供逐条审批（简化，不做复杂审核流）

## 4. 关键状态流转图

```
                        家长配置今晚计划
                              │
                              ▼
                    ┌─────────────────────┐
                    │    开始前 (Pre-Start)   │
                    │  "今晚计划将于 xx:xx 开始"  │
                    │  可预览任务格子，计时未启动   │
                    └──────────┬──────────┘
                               │ 时间到达起始点
                               ▼
                    ┌─────────────────────┐
                    │    进行中 (In Progress)   │
                    │  实时倒计时 + 任务格子       │
                    │  自由时间发光 + 紧迫感反馈    │
                    │  孩子可点击完成             │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
              ▼                ▼                ▼
     ┌────────────────┐ ┌────────────────┐ ┌────────────────┐
     │ 全部完成 (All Done) │ │ 进行中（继续执行）│ │ 未完成超时的孩子  │
     │ "任务已完成，      │ │                │ │ 进入结束未完成   │
     │  剩余时间自由安排"  │ │                │ │ 状态            │
     └────────────────┘ └────────────────┘ └────────────────┘
                               │                │
                               │                ▼
                               │        ┌────────────────┐
                               │        │ 结束后未完成     │
                               │        │ (Ended-Incomplete)│
                               │        │ "已超出计划时间"  │
                               │        │ 保留未完成任务    │
                               │        └────────────────┘
                               │                ▲
                               │                │
                               └────────────────┘
```

### 状态说明表

| 状态 | 名称 | 时间条件 | 时间条状态 | 任务状态 | 自由时间 |
|------|------|----------|-----------|---------|---------|
| S1 | 开始前 | < 起始时间 | 未消耗，可预览 | 全部 pending | 预估发光（虚） |
| S2 | 进行中 | 起始时间 ≤ now ≤ 结束时间 | 实时消耗 | pending / done 混合 | 实际发光 |
| S3 | 结束后全部完成 | now > 结束时间 且任务全部 done | 推进到末端 | 全部 done | 全为自由时间 |
| S4 | 结束后未完成 | now > 结束时间 且存在 pending | 推进到末端 | 存在 pending | 剩余部分发光 |

### 异常边界状态

| 场景 | 界面表现 |
|------|---------|
| 无任何任务配置 | 显示"今晚暂无任务安排" |
| 任务总时长 > 时间窗口 | 保存允许，但后台给出提示 |
| 孩子误点完成 | 家长在进度监控页撤销最近一次，时间条恢复 |
| 时间窗口未到但全部完成 | 显示"今晚任务已完成，剩余时间自由安排" |

## 5. 信息层级建议

### 5.1 孩子端信息优先级

```
第1层（最大最显眼）：当前下一步任务 + 剩余时间
第2层（次要显眼）：   主时间条（含自由时间发光区）
第3层（辅助）：       其他待完成任务列表
第4层（底部）：       任务完成操作入口
```

### 5.2 家长端信息优先级

```
第1层（默认打开）：今晚计划页（起止时间 + 任务列表）
第2层（次要）：     孩子管理页（档案编辑）
第3层（按需）：     进度监控页（纠偏撤销）
```

### 5.3 孩子端 vs 家长端职责对比

| 维度 | 孩子端 | 家长端 |
|------|--------|--------|
| 主要操作 | 看 + 点完成 | 配 + 改 + 撤销 |
| 能看到谁的任务 | 仅自己 | 全部孩子 |
| 能修改任务吗 | ❌ 不能 | ✅ 可以 |
| 能撤销完成吗 | ❌ 不能 | ✅ 可以（纠偏） |
| 能看到自由时间吗 | ✅ 可以（核心反馈） | ✅ 可以（但非核心） |
| 能编辑孩子档案吗 | ❌ 不能 | ✅ 可以 |
