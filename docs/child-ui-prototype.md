# Issue #4 — 孩子主界面高保真原型方向

## 概述

本文档定义 iPad 家庭晚间任务时间面板的孩子端主界面原型方向，聚焦于时间条、任务格子、当前下一步任务提示与自由时间反馈的视觉表达。

输入来源：`docs/prd.md`、`docs/research.md`
设计原则：时间条优先于清单；风格亲和可爱；紧迫感可爱但不惊吓；支持 4 个孩子切换。

### 布局层级说明

| 层级 | 元素 | 说明 |
|------|------|------|
| 顶部 | 孩子切换栏 | 4 个头像/昵称按钮，当前孩子高亮选中 |
| 中上 | 时间窗口条 | 今晚起止时间 + 总体倒计时 |
| 中 | 主时间条 | 实时消耗，时间轴从左到右缩短 |
| 中下 | 当前任务提示卡 | 最强视觉权重，比任务格子更突出 |
| 下 | 任务格子区 | 所有任务按顺序排列 |
| 底部 | 自由时间反馈 | 发光区域，提示已赢得多少可玩时间 |

## 二、顶部孩子切换栏

### 视觉

- 4 个圆形头像（或颜色块 + 昵称），横向排列于顶部
- 当前选中孩子：头像外圈加粗发光 + 底部出现细线高亮
- 未选中孩子：灰度较低，但仍可点击切换
- 切换时整体面板内容刷新为此孩子的任务

### 尺寸建议

- 头像直径：56–64px
- 间距：12px
- 顶部留 16px 安全边距

## 三、主时间条（核心视觉）

### 结构

```
[已消耗████████]|[当前任务]|[未完成任务]|[▓▓自由时间▓▓]
```

- **已消耗段**：深灰色或该孩子代表色，视觉压缩感
- **当前任务段**：高亮强调色，呼吸式脉冲动效（轻微，不是警报）
- **未完成任务段**：标准任务色，比当前任务段略淡
- **自由时间发光段**：荧光蓝/绿 + 柔和光晕，明确区别于任务区

### 宽度映射

- 时间条宽度 = 时间窗口总长（如 5:30–20:30 = 3 小时）
- 每个任务格子宽度 = 该任务预计时长 / 总窗口 × 100%
- 实时缩短以"剩余时间占总窗口比例"计算

### 状态差异

| 状态 | 主时间条表现 |
|------|------------|
| 开始前 | 时间条灰色占位，文字提示"今晚 5:30 开始" |
| 进行中 | 正常消耗，实时缩短 |
| 时间紧迫 | 整体条微微发暖色（橙调），未完成格子轻微膨胀 |
| 超时未完成 | 条右侧溢出显示，时间条变红橙色，文字提示"已超出" |

## 四、当前下一步任务高亮方案

### 核心思路

当前任务不是"清单里的一项"，而是一个需要孩子立即行动的强提示。视觉权重应明显高于其他任务格子。

### 高亮方案

```
┌─────────────────────────────────────────────────────┐
│  ▶  现在做：吃饭                    [完成了 ✓]     │
│  ╭─────────────────────────────────────────────╮   │
│  │  剩余 20 分钟 · 快做完可以多玩一会儿！        │   │
│  ╰─────────────────────────────────────────────╯   │
└─────────────────────────────────────────────────────┘
```

**设计要素：**

- 卡片大小：宽度撑满，厚度感（轻微投影 + 圆角 16px）
- 背景色：该任务对应颜色的 120% 亮度 + 微妙渐变
- 左侧三角指示箭头动画（缓慢呼吸）
- 右侧完成按钮明显但幼儿友好（圆形，大面积点击）
- 辅助文案（可选，底部细字）："快做完可以多玩一会儿！"——增强动力感

### 未开始前的下一个任务

- 非当前任务，但即将到来
- 显示为半透明预告状态（任务格子带虚线边框）
- 标注"接下来：xxx"

## 五、任务格子区

### 单个任务格子

```
┌──────┐
│ 🛁   │ 洗澡
│ 15'  │ 剩余 15 分钟
└──────┘
```

- 左侧图标（或 Emoji）
- 中间任务名称
- 底部剩余时长（实时）
- 圆角：12px
- 间距：8px

### 三种任务状态

| 状态 | 视觉表现 |
|------|---------|
| 未开始 | 正常颜色，灰底 |
| 进行中 | 当前任务高亮卡片（见上节），不在格子里重复强调 |
| 已完成 | 透明度 40%，打勾图标，灰色文字 |
| 超时（紧迫） | 格子轻微膨胀 + 橙色边框 + 轻微抖动动画 |

## 六、自由时间发光与紧迫感反馈

### 自由时间——发光视觉表达

**目标：** 让孩子一眼理解"这是我快速完成任务换来的可玩时间"。

```
████████████████████████░░░░░░░░░░░░░░░
                         ↑ 荧光发光区域
```

**视觉特征：**

- 使用荧光蓝 / 青绿色（`#00E5FF` 或类似）
- 底部有柔和光晕（box-shadow blur，不要硬边）
- 可加小星星 ✨ 装饰符号点缀
- 文字提示：`已获得自由时间 1h 12m ✨`

**动态行为（第一版 MVP）：**

- 任务完成后，对应格子消失，自由时间区域在时间条上"长出"
- 已有自由时间被新完成任务撑大时，发光面积扩大

### 紧迫感——可爱但不惊吓

**目标：** 传达"时间不够了"，但不要做成警报、红色大字、警告音效。

**视觉手段：**

| 手段 | 表达 |
|------|------|
| 色温变化 | 主时间条整体向暖橙偏移（不是红色） |
| 格子膨胀 | 未完成任务格子轻微放大 5–10% |
| 呼吸感 | 未完成格子每 3s 微微亮一下（比普通状态更亮一点） |
| 背景色 | 主面板背景从冷色调向暖橙微微渐变（可选项，第一版可跳过） |
| 文案提示 | "要抓紧啦！" ——友好文案，非警告语气 |

**禁止：**

- ❌ 红色背景块
- ❌ 大字"超时！"
- ❌ 连续闪烁
- ❌ 音效或震动提示

## 七、三种状态界面差异

### 状态 1：开始前（待机）

```
┌─────────────────────────────────────────────────────┐
│  [🟢小明] [🟡小红] [🔵小华] [🟣小丽]                │
│                                                     │
│  🌙 今晚安排将于 5:30 开始                          │
│                                                     │
│  预览：                                            │
│  🛁 洗澡  🟡 ☐ 作业  🟡 ☐ 自由时间                  │
│                                                     │
│  还有 1 小时 23 分钟                                │
└─────────────────────────────────────────────────────┘
```

- 不进入倒计时消耗模式
- 时间条以淡色占位，提示即将开始
- 可预览今晚任务顺序
- 底部显示"还有 X 小时 X 分钟"

### 状态 2：进行中

见前述布局。正常消耗，显示当前任务、自由时间。

### 状态 3：结束后未完成

```
┌─────────────────────────────────────────────────────┐
│  [🟢小明] [🟡小红] [🔵小华] [🟣小丽]                │
│                                                     │
│  😟 已超出计划时间                                  │
│                                                     │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━        │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ 超时    │
│                                                     │
│  还剩：                                            │
│  ☐ 作业  ☐ 运动                                   │
│                                                     │
│  家长可在后台调整今晚安排                           │
└─────────────────────────────────────────────────────┘
```

- 时间条右侧溢出（超时部分）
- 未完成任务仍在格子里显示
- 文案语气柔和，不指责
- 不直接消失或留空白

## 八、4 个孩子切换的视觉策略

### 切换入口

顶部头像/昵称切换条（默认方案）

### 切换后界面刷新

- 孩子切换时：时间条重算 + 任务列表刷新 + 自由时间重置
- 视觉过渡：轻微淡入淡出（150ms），不需复杂动效

### 每个孩子的颜色标识

| 孩子 | 代表色 |
|------|-------|
| 小明 | 绿色 `#4CAF50` |
| 小红 | 黄色/橙色 `#FF9800` |
| 小华 | 蓝色 `#2196F3` |
| 小丽 | 紫色 `#9C27B0` |

- 孩子颜色用于：头像边框、当前任务卡片背景、切换选中态

## 九、验收标准对照

| 验收标准 | 实现方式 |
|---------|---------|
| 能一眼看出当前时间、下一步任务与自由时间 | 主时间条三段式结构 + 当前任务高亮卡片 + 底部自由时间发光区 |
| 三种状态下的视觉差异清楚 | 开始前灰度占位 / 进行中正常消耗 / 超时暖色溢出 + 未完成格子保留 |
| 用户可基于原型继续拍板视觉方向 | 文字描述已完整，可转化为 Figma/Sketch 画布 |
